# Чеклісти для тестування

## Модуль: Відображення розкладу для обраної групи (публічна частина)

### Функціональність (позитивні сценарії)

- [ ] Користувач може обрати групу із випадаючого меню чи почати друкувати номер групи в пошуку
- [ ] Після вибору групи завантажується та відображається розклад саме цієї групи
- [ ] Розклад чітко розділений на непарний та парний тижні (таблиця для непарного знаходиться нижче таблиці для парного)
- [ ] В першому (лівому) стовбці підписано номер пари та вказано час початку та закінчення пари
- [ ] У розкладі відображаються всі дні тижня (Пн–Нд), навіть якщо в якомусь дні немає занять
- [ ] У комірках правильно відображається: назва дисципліни, викладач, аудиторія, тип заняття (Лекція/Практичне/Лабораторна)

### Негативні сценарії та крайні випадки

- [ ] Обрано групу, у якої взагалі немає розкладу (очікується: «Розклад відсутній»)
- [ ] Спроба обрати неіснуючу групу (через пряме введення в URL) виводить "Schedule is empty"
- [ ] Швидке перемикання між кількома групами поспіль: розклад не «зависає», не показує дані попередньої групи
- [ ] Розклад містить заняття без вказаного викладача/аудиторії/заняття, тоді комірка відображається без зміни розміру комірки таблиці, пропущені поля відсутні або замінені тире/пробілом

### UI/UX та зручність

- [ ] Назва обраної групи чітко відображається над розкладом в меню
- [ ] Розмір шрифту в комірках дозволяє комфортно читати з телефону (≥14–15 px на мобільному)
- [ ] На мобільних пристроях таблиця/сітка не виходить за межі екрану горизонтально (адаптивність)
- [ ] Візуально зрозуміло, де закінчується один день і починається інший (чіткі межі стовпців)
- [ ] Теперішній день тижня виділено блакитним фоном
- [ ] Над таблицями розкладу відображається підпис (парний чи непарний тиждень)
- [ ] Теперішній тиждень (парний чи непарний) виділено блакитним фоном
